Layer 3 — Orchestrator Agent
🧠 Orchestrator Agent backend/agents/orchestrator.py
Step 1 — Symbol Normalisation
  • btc / eth / sol → BTCUSDT / ETHUSDT / SOLUSDT delta
  • reliance / tcs / nifty → NSE:RELIANCE-EQ fyers
  • banknifty / sensex → NSE:BANKNIFTY / BSE:SENSEX fyers
Step 2 — Intent Routing (LLM)
User says Intent Route
"Analyse BTCUSDT" analyse → Analysis Agent
"Execute trade on ETH" execute → Analysis + Execution Agent
"What is Canara profit?" research → Tavily MCP → LLM
"Hello / general Q" chat → LLM directly
A2A Clients
  • Analysis client → http://localhost:8101 (timeout 300s)
  • Execution client → http://localhost:8102 (timeout 300s)
  • Catches ConnectError + TimeoutException → failed TaskResponse
⚠️ In-process fallback: if :8101 is unreachable or times out, AnalysisAgent runs directly inside :8100 — analysis still works with only one service running.
LLM Factory
  • OpenAI → gpt-4o-mini
  • Ollama → local model (http://localhost:11434)
  • Injected per-request, no global state

Layer 4 — Specialist Agents
🔬 Analysis Agent backend/agents/analysis_agent.py · :8101
What it does
  • Receives: symbol, source, trade_type, llm_provider
  • Fetches candles + metrics + news in parallel (asyncio.gather)
  • Builds prompt with OHLCV data + funding rates + headlines
  • Calls LLM → structured JSON analysis
Output schema
  • trend · strength · last_price
  • entry_zone · stop_loss · targets[]
  • rr_ratio · confidence · summary
  • key_levels · news_context
MCP Tools used
  • delta fetch_candles
  • delta fetch_perpetual_metrics
  • fyers fetch_candles
  • fyers get_quote
  • tavily search_news
intraday swing longterm
⚡ Execution Agent backend/agents/execution_agent.py · :8102
What it does
  • Receives: analysis output + mode (paper/real)
  • Calls LLM → generates execution plan
  • Paper mode: simulates order, returns PAPER-{task_id}
  • Real mode: places live bracket order via MCP
Output schema
  • action · symbol · broker
  • order_type · entry · stop_loss
  • take_profit · qty · risk_amount
  • rr_ratio · rationale · mode
MCP Tools used
  • delta place_order
  • delta place_market_order
  • fyers place_order
  • fyers get_positions
⚠️ Real trades place live bracket orders. Toggle Paper/Real carefully.

Layer 5 — MCP Tool Servers (auto-spawned subprocesses)
🔷 Delta Exchange MCP mcp_servers/delta_server.py
  • fetch_candles — OHLCV data (any interval)
  • fetch_perpetual_metrics — funding rate, OI, mark price
  • get_mark_price — current LTP
  • place_order — bracket order (authenticated)
  • place_market_order — simple market order
  • get_positions — open positions
  • get_orders — order history
  • cancel_order — cancel by order ID

  • fetch_candles — NSE/BSE OHLCV data

Full Data Flow — Analyse Request
Browser  →  POST /chat { message: "Analyse BTCUSDT for intraday" }
   Gateway :8100  →  OrchestratorAgent.handle_message()
      Orchestrator  →  LLM: classify intent & symbol → { intent: "analyse", symbol: "BTCUSDT", source: "delta" }
      Orchestrator  →  A2A POST http://localhost:8101/a2a/task
         Analysis Agent :8101  →  asyncio.gather(
                                 delta.fetch_candles(BTCUSDT, 5m),
                                 delta.fetch_perpetual_metrics(BTCUSDT),
                                 tavily.search_news("BTCUSDT analysis")
                            )
         Analysis Agent  →  LLM(OHLCV + funding rate + news) → JSON analysis
      Orchestrator  →  format reply + return { reply, analysis }
   Gateway  →  save messages to SQLite, count tokens
Browser  ←  { reply, analysis: { trend, entry_zone, stop_loss, targets, rr_ratio, ... } }
Chart  reloads to BINANCE:BTCUSDT at selected timeframe
